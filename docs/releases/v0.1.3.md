# Release Notes - v0.1.3

openai-ralph-codex v0.1.3 turns the recent Ralph loop work into a more complete product surface. This release adds compact task contracts, latched continuation routing, distilled loop memory, value/cost scheduling, and deterministic split proposals so Ralph can stay multilingual, stateful, and token-efficient without forcing users onto cheaper models.

## Highlights

- Codex stage-classifier entry plus loop-session latch for smaller continuation routing
- Failure fingerprints instead of replaying raw failure logs into later turns
- Compact task prompt packs with acceptance criteria and verification hints
- Task-aware verification profile fallback in `ralph run` and `ralph verify`
- Distilled project memory under `.ralph/memory.json`
- Value/cost runnable-task scoring in the scheduler
- Auto-split proposals for context-budget blocks and repeated failures

## What's Changed

### Routing and loop continuity
- classifier-led hook entry remains the only loop trigger path
- active Ralph loops now use a shorter continuation classifier prompt
- loop-session metadata is persisted in `.ralph/state.json`

### Task contracts and prompt hygiene
- planner now compiles acceptance criteria and verification hints onto tasks
- runner prompts are built from compact task packs instead of looser inline text
- task-local verification hints can drive verification when the project has no explicit global commands

### Recovery and memory
- failures now persist as typed fingerprints instead of relying on raw log replay
- distilled reusable lessons are stored in `.ralph/memory.json`
- blocked broad work now generates `.ralph/split-proposals.json`

### Throughput
- runnable tasks are scored by likely value versus cost instead of simple source order
