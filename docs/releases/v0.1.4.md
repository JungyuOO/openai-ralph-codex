# Release Notes - v0.1.4

openai-ralph-codex v0.1.4 adds adaptive prompt modes to the Ralph runner so compactness is no longer one-size-fits-all. Small clean tasks now get a lighter prompt, while retry-heavy or broad-risk tasks keep richer recovery context. This preserves quality better than an aggressively uniform prompt shrink strategy while still reducing unnecessary prompt weight.

## Highlights

- Adaptive runner prompt modes: `small`, `balanced`, and `recovery`
- Internal-only heuristics for prompt shaping; no new user-facing threshold settings
- Smaller prompts for low-risk narrow tasks
- Richer recovery prompts retained for failed or broad-risk tasks
- README updated to describe the shipped product behavior more accurately

## What's Changed

### Prompt behavior
- `src/core/prompt-pack.ts`
  - added internal adaptive prompt-mode selection
  - `small` mode trims acceptance criteria, notes, and memory more aggressively for clean low-load tasks
  - `recovery` mode keeps more contract detail for tasks with failure or broad-risk signals
  - `balanced` remains the default middle lane
- `tests/core/prompt-pack.test.ts`
  - added regression coverage for prompt-mode selection and mode-specific prompt shaping

### Docs
- `README.md`
  - documents adaptive prompt modes as a shipped product behavior
