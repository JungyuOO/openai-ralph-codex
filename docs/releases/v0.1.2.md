# Release Notes - v0.1.2

openai-ralph-codex v0.1.2 removes keyword-list trigger routing from the Ralph hook and makes Codex's stage classifier the source of truth for entering the Ralph loop. This keeps multilingual routing centered on model intent judgment instead of hand-maintained signal tables.

## Highlights

- Codex stage-classifier routing is now the only hook trigger path
- Removed keyword/rule-list heuristics for planning, execution, verify, resume, and status detection
- Legacy sync helper exports now stay inert instead of reintroducing keyword-trigger behavior
- Added regression coverage for classifier-unavailable and heuristic-disabled paths
- Prepared the next package release as `openai-ralph-codex@0.1.2`

## What's Changed

### Changed
- `plugins/openai-ralph-codex/scripts/ralph-hook.mjs`
  - removed keyword-signal scoring tables and heuristic stage selection from hook routing
  - `determineStage()` now returns classifier-driven decisions only
  - when no classifier decision is available, hook routing safely returns `ignore`
- `plugins/openai-ralph-codex/scripts/ralph-hook.d.ts`
  - aligned route-decision typing with classifier/guard-only sources
- `tests/plugins/plugin-hooks.test.ts`
  - replaced heuristic-routing expectations with classifier-only regression coverage

### Release prep
- bumped package version to `0.1.2`
- added a repo-local release-note rule to `AGENTS.MD`
