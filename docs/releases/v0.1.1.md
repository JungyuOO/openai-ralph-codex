# Release Notes - v0.1.1

## Highlights

- Published `openai-ralph-codex@0.1.1` to npm
- Global install now targets a no-clone workflow
- Home-local plugin install now updates:
  - `~/plugins/openai-ralph-codex`
  - `~/.agents/plugins/marketplace.json`
  - `~/.codex/hooks.json`
- First relevant prompt can now lazily bootstrap `.ralph/` inside a brand-new project
- Prompt routing is stronger for:
  - PRD / planning
  - execution
  - verification
  - blocked / resume flows

## Install

```bash
npm install -g @openai/codex openai-ralph-codex
```

## Expected user flow

1. Install the package globally
2. Open any project
3. Run `codex`
4. Ask for PRD planning, feature work, verification, or blocked-work continuation
5. Ralph hooks surface the right workflow
6. If `.ralph/` does not exist yet, Ralph bootstraps it lazily

## Included capabilities

- `ralph init`
- `ralph plan`
- `ralph run`
- `ralph verify`
- `ralph resume`
- `ralph status`
- `ralph plugin install`
- `ralph plugin status`

## Known limits

- Ralph is still command-driven for the execution lane
- Hooks auto-surface and bootstrap, but do not yet auto-run nested `ralph run`
- Real Codex host behavior was smoke-tested through the package and hook path, but not every Codex host variant has been exhaustively validated
