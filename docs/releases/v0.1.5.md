# Release Notes - v0.1.5

openai-ralph-codex v0.1.5 makes project opt-in the default safety model for Codex hook routing. Global install now prepares the plugin infrastructure, but each repository must explicitly opt in with `orc enable`. This release also promotes the new `orc` command surface as the primary user-facing CLI while keeping `ralph` as a compatibility alias.

## Highlights

- Project-scoped activation via `orc enable` and `orc disable`
- New `orc status --project` command to inspect activation before bootstrapping
- Hook routing now exits unless the current repo has `.ralph/project.json`
- `orc` promoted as the primary CLI name in docs and user-facing guidance
- Release metadata bumped to v0.1.5 across the package and plugin manifest

## What's Changed

### Project activation
- `src/commands/enable.ts`
  - adds an explicit project opt-in command that writes `.ralph/project.json`
- `src/commands/disable.ts`
  - adds an explicit project opt-out command that removes the activation marker
- `src/core/project-activation.ts`
  - centralizes activation read/write/remove behavior
- `src/schemas/project-activation.ts`
  - validates activation marker shape before it is trusted

### Status and routing UX
- `src/commands/status.ts`
  - adds `--project` support so activation can be inspected before `.ralph/state.json` exists
- `src/cli.ts`
  - wires `enable`, `disable`, and `status --project`
  - publishes `orc` as a first-class CLI alias
- `plugins/openai-ralph-codex/scripts/ralph-hook.mjs`
  - gates hook routing on `.ralph/project.json`
  - updates recommended command paths to the `orc ...` surface

### Release polish
- `README.md`
  - documents project opt-in flow and `orc`-first command examples
- `plugins/openai-ralph-codex/.codex-plugin/plugin.json`
  - bumps plugin manifest version to `0.1.5`
- `package.json`
  - bumps npm package version to `0.1.5`
